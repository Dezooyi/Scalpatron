Trading-Strategie

Range Spike Scalper — nutzt repetitive Preis-Spikes innerhalb einer Konsolidierungszone.

Grundidee

Spike Peak (SELL) ╱╲ ╱ ╲ ╱ ╲──── Drop ≥ sellDropThreshold → Sell-Signal ╱ ╲ ────╱────────╲────────── Stufenboden (Floor = Median) ▲ ▲ BUY BUY (spike% ≥ threshold)

Signale

SignalBedingungAktion
BUYSpike% ≥ spikeThreshold, kein aktiver Spike, kein CooldownKaufe UGOR mit tradeSize SOL
SELLIm Spike, Drop vom Peak ≥ sellDropThresholdVerkaufe gesamte Position → SOL
HOLDAlles andere (Datensammlung, Cooldown, kein Spike)Nichts tun

Floor-Berechnung

Median der letzten floorWindow Preise. Median statt Durchschnitt weil:

  • Robust gegen einzelne Spike-Ausreißer
  • Floor bleibt stabil auch während Spikes

State-Machine

Spike% ≥ Threshold ┌──────────┐ ──────────────────▶ ┌──────────────┐ │ WAITING │ │ IN SPIKE │ │(kein Pos)│ ◀────────────────── │(Peak tracken)│ └──────────┘ Drop ≥ sellDrop └──────────────┘ ▲ │ │ ▼ ┌──────────┐ │ COOLDOWN │ (N Ticks warten) └──────────┘

Risiken

  • Kein Slippage-Modell im Paper-Modus
  • Polling-Latenz (2s) — echte Spikes können kürzer sein
  • UGOR-Liquidität begrenzt — große Orders bewegen den Preis
  • MEV/Frontrunning auf Solana möglich

Preis │ │ Peak ─────┐ │ ╱ │ ← sellDropThreshold (0.15%) │ ╱ ▼ │ ╱ Spike Sell-Signal │ ╱ │────╱───── spikeThreshold (0.3%) ── BUY-Signal │ │═══════════════ Floor (Median) │ └────────────────────────────── Zeit ◀──────────▶ floorWindow
